# OpenClaw Integration

## Current Status

`Koinara-node` now supports a built-in `openclaw` provider backend.

That means you can configure the node so that:

- `Koinara-node` keeps control of job polling, response hashing, wallet signing, submission, verification, and reward settlement
- `OpenClaw` produces the provider-side inference text through the local OpenClaw CLI

This is the clean split:

- `OpenClaw` = agent workflow and inference source
- `Koinara-node` = protocol execution boundary

## What Exists Now

Today, you can use OpenClaw in two practical ways.

### 1. OpenClaw as an operator layer

Use OpenClaw for:

- prompt planning
- response drafting
- verifier-side review assistance
- job inspection and workflow orchestration

In this shape, OpenClaw helps the operator, while `Koinara-node` still runs as the actual provider or verifier runtime.

### 2. OpenClaw as a built-in provider backend

Use this when you want provider submissions to be generated by an OpenClaw agent.

Minimal config shape:

```json
{
  "provider": {
    "backend": "openclaw",
    "supportedJobTypes": ["Simple"],
    "openclaw": {
      "agent": "main",
      "local": true,
      "thinking": "low",
      "timeoutSeconds": 120
    }
  }
}
```

At runtime the node will:

1. poll open jobs
2. load the manifest
3. pass the manifest prompt to OpenClaw
4. store the output artifact
5. compute the canonical `responseHash`
6. submit on-chain as the provider wallet

## Minimal Verification Command

Before using the OpenClaw backend in production, confirm the local agent works:

```bash
openclaw agent --agent main --local --json --message "Reply with exactly OK"
```

If that succeeds, the node can use the same local OpenClaw CLI path for provider inference.

## What OpenClaw Alone Still Cannot Do

OpenClaw by itself is still not enough to participate as a Koinara provider or verifier.

Without a running `Koinara-node`, an OpenClaw agent does not:

- monitor Koinara job state on-chain
- compute and submit the canonical `responseHash`
- call `submitResponse`
- call verifier actions such as `verifySubmission`, `rejectSubmission`, or `finalizePoI`
- claim `active` or `work` rewards after an epoch closes

## Minimal Local Architecture

```text
OpenClaw Agent
  -> receives the prompt from Koinara-node
  -> produces provider-side inference text

Koinara-node
  -> polls jobs
  -> validates manifests and schemas
  -> stores receipts and artifacts
  -> signs transactions
  -> submits provider / verifier actions
  -> claims rewards after epoch close
```

## Recommended Setup Today

1. Install and configure `Koinara-node`
2. Confirm `openclaw agent --agent main --local --json` works locally
3. Set `provider.backend = "openclaw"`
4. Run:

```bash
npm run provider:v2:doctor
npm run provider:v2:start
```

5. After an epoch closes, claim with:

```bash
npm run provider:v2:claim
```

If you also run a verifier:

```bash
npm run verifier:v2:start
npm run verifier:v2:claim
```

## What Does Not Exist Yet

The repository now ships:

- a workspace-ready OpenClaw skill at `skills/koinara-node/SKILL.md`
- install helpers:
  - `scripts/install-openclaw-skill.ps1`
  - `scripts/install-openclaw-skill.sh`

What still does not exist yet:

- a first-class `openclaw connect koinara-node` command
- a MetaMask-style browser signer bridge for the node runtime

Those remain future convenience layers above the public node.

## Install the OpenClaw Skill

If you want the Koinara skill available in your global OpenClaw skills folder:

### Windows PowerShell

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\install-openclaw-skill.ps1
```

### macOS / Linux

```bash
bash ./scripts/install-openclaw-skill.sh
```

This copies the skill into:

- Windows: `C:\Users\<user>\.openclaw\skills\koinara-node`
- macOS / Linux: `~/.openclaw/skills/koinara-node`

If you open the repository itself as an OpenClaw workspace, the skill can also be loaded directly
from the repo-local `skills/` folder.

## Direction

The intended long-term shape is:

- OpenClaw for agent workflow, delegation, and coordination
- `Koinara-node` for chain-facing execution
- optional skill packages or dashboards on top of both

That keeps the public node simple, while making OpenClaw a natural agent layer on top of Koinara.
